# Google Cloud Maps Integration Prompt

## Task: Connect Geolocation-Based Real-Time Map via Google Cloud Console

### Overview
Integrate Google Maps API from Google Cloud Console instead of Leaflet/OpenStreetMap to display real-time geolocation-based maps with custom markers for all legend items.

### Required Setup Steps

1. **Google Cloud Console Setup**
   - Create a new project in Google Cloud Console
   - Enable the following APIs:
     - Maps JavaScript API
     - Geolocation API
     - Maps Geocoding API
   - Create an API key with Web restrictions
   - Add the API key to `.env` file as `GOOGLE_MAPS_API_KEY`

2. **Map Configuration**
   - Initialize Google Maps instance centered on Assam region (26.5°N, 92.8°E)
   - Set zoom level to 9
   - Apply dark/light theme styling based on app theme toggle
   - Enable user geolocation detection with browser's Geolocation API

3. **Legend Markers to Display**
   - **Blue Marker**: User's Current Location (auto-detected GPS)
   - **Red Circles**: High-Risk Flood Zones
     - Dibrugarh: 26.47°N, 94.91°E (radius: 20km)
     - Jorhat: 26.75°N, 94.20°E (radius: 25km)
   - **Yellow Circles**: Flood Watch Areas
     - Sibsagar: 26.95°N, 94.55°E (radius: 18km)
   - **Green Markers**: Active Shelters
     - Guwahati-5 (St. Xavier): 26.2°N, 91.74°E
     - Guwahati-3 (District Hall): 26.15°N, 91.71°E
     - Jorhat Center: 26.75°N, 94.20°E
     - Dibrugarh Relief: 26.47°N, 94.91°E

4. **Implementation Details**

   **HTML Structure**
   ```html
   <script src="https://maps.googleapis.com/maps/api/js?key=YOUR_API_KEY&libraries=geometry,drawing"></script>
   <div id="map-container" style="height: 500px; width: 100%;"></div>
   ```

   **JavaScript Features to Implement**
   - Auto-detect user location on page load
   - Add InfoWindows to all markers showing location details
   - Pan/zoom to user location automatically
   - Color-code markers based on legend
   - Add real-time updates capability
   - Legend control in bottom-right corner
   - Responsive sizing for different screen sizes
   - Dark/light mode support matching app theme

5. **Pages to Update**
   - Overview (index.html)
   - Community Reports (community-reports.html)
   - Evacuation (evacuation.html) - Optional

6. **Customization Options**
   - Custom marker icons using Google's Advanced Markers
   - Heatmap layers for flood intensity visualization
   - Polygon overlays for zone boundaries
   - Real-time traffic and weather overlays
   - Street View integration for location preview

### Important Notes
- Ensure API key security (use environment variables, not hardcoded)
- Implement rate limiting to avoid exceeding free tier quotas
- Cache geolocation results appropriately
- Use Google's Geocoding API for reverse geocoding addresses
- Test in both light and dark themes
- Ensure responsive design for mobile devices

### Environment Variables Required
```
GOOGLE_MAPS_API_KEY=your_api_key_here
GOOGLE_MAPS_ENABLED=true
```

### Fallback
- Keep Leaflet/OpenStreetMap as fallback if Google Maps fails to load
- Implement error handling for API failures
